DigSig-ng - ELF signature verification in the Linux 3.x kernel

Introduction

DigSig-ng is a Linux kernel security module, providing runtime support for RSA digital signature verification of ELF executables and dynamic libraries. It is intended to increase system security by only allowing trusted ELF executables and libraries to run.

DigSig-ng is a fork of the DigSig project, which began in 2003 as part of a research project by Ericsson Research Canada (Open Systems Lab), but has been unmaintained since March 5th 2009, with the last major release on January 29th 2008.

DigSig-ng aims to maintain the DigSig project for newer kernels, but also to improve the original project in several ways. DigSig-ng's future roadmap aims to provide support for newer, more secure cryptographic hash functions (i.e. SHA-2 and SHA-3), support for RSA key lengths above 2048 bits and the ability to stack DigSig with other kernel security modules - see the Roadmap section for a more complete list.

Feature Summary

  • Kernel-mode digital signature verification for ELF executables and dynamic libraries
  • Uses SHA-1 digest and RSA signature, with key length of 1024 or 2048 bits (to be expanded, see below)
  • Signature validation caching, negating the performance overhead of signature verification
  • Signature revocation
  • Mitigation against TOCTTOU attacks via USB mass storage devices or network filesystems (with caveats)
  • Implemented as a Linux Security Module, currently building against Linux 3.12

Roadmap

  • Support for arbitrarily sized (3072-bit and longer, if desired) RSA keys
  • Support for signature verification using elliptic curve cryptography (ECDSA)
  • SHA-256 and SHA-512 (and exploring the possiblity of SHA-3) digest algorithm
  • dsngctl userspace tool for tuning module parameters (work in progress at https://github.com/digsig-ng/dsngctl.git repository)
  • dsngsign userspace tool for digitally signing ELF binaries
  • Ability to stack with other Linux security modules
  • Support for other kernel releases (particularly, support for every LTS kernel release, plus the latest kernel release)

Develop

Development of the DigSig-ng project is conducted using git for revision control, with repositories currently hosted on GitHub. You can check out the kernel sources at https://github.com/digsig-ng/linux-digsig.git. The DigSig-ng source code lives within the security/digsig directory of the source tree.

You can obtain the sources for dsngctl from https://github.com/digsig-ng/dsngctl.git, which supersedes the extract_pkey tool available in the digsig source tree.

The bsign tool might not be packaged for your distribution, but you can download and compile the source code using autotools from the mirror at https://github.com/digsig-ng/bsign-mirror.git. bsign is necessary to sign binaries.

Breaking Issues

There are some known issues which are currently being worked on. These issues include:

  • Attempting to compile while CONFIG_MPILIB is enabled will fail due to a conflict between the module's gnupg version and the gnupg MPI library present in the Linux kernel

Documentation

There is currently no documentation available as part of the DigSig-ng project, however, the documentation on the DigSig/DSI website is still applicable. Documentation for DigSig-ng will hopefully be prepared in time for the first major release.
